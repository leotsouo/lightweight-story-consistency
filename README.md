## 輕量智敘：長篇敘事一致性檢核系統（前端 Prototype）

輕量智敘是一套針對**長篇文本（劇本、小說、企劃書等）敘事一致性檢核**而設計的產品雛型，  
本專案為其 **前端展示介面（Frontend Prototype）**，以 **Next.js + Tailwind CSS** 實作，  
聚焦於產品體驗與資訊架構呈現，示範系統如何：

- 協助敘事設計師與編輯快速掌握角色、事件與世界觀之間的邏輯關係
- 以視覺化介面呈現衝突與風險點，支援後續修訂決策
- 將未來可串接的輕量化 LLM / 邊緣部署能力，以易用的工具形態封裝給創作者使用

> 說明：目前專案定位為 **展示用 Demo / Prototype**，尚未串接真實後端與模型，  
> 所有檢核結果與統計資料皆為 **mock data**，用於模擬實際使用情境。

---

### 功能與頁面總覽

- **Dashboard 首頁**
  - 專案總覽、最近檢核結果
  - 衝突數量摘要（角色 / 事件 / 規則等）
  - 角色數 / 事件數 / 規則數
  - 最近版本比較摘要

- **專案／文本上傳頁**
  - 上傳劇本／小說／企劃文本
  - 版本資訊顯示
  - 檔案狀態與分析進度（以 mock 狀態條呈現）

- **檢核結果頁**
  - 整體衝突總數
  - 依類別的衝突卡片（角色／時間／物件／世界觀等）
  - 衝突列表，可點擊展開前後文片段

- **角色一致性頁**
  - 角色清單與角色設定摘要
  - 各章節的行為／能力／性格變化
  - 可能「角色漂移」的警示標示

- **知識圖譜頁**
  - 以圖像化方式呈現角色、物件、事件、地點的關聯
  - 可點擊節點檢視詳細資訊（mock 視窗／側欄）

- **報告頁**
  - 顯示衝突彙整與建議修正方向
  - 提供模擬的「匯出 PDF 報告」按鈕（無實際匯出邏輯）

---

### 目標使用者與應用情境

- **目標使用者**
  - 遊戲敘事設計師 / 關卡設計團隊
  - 影視編劇與企劃團隊
  - 出版社編輯、原創 IP 團隊

- **典型情境**
  - 長篇作品歷經多次改稿後，需快速檢查：  
    - 人物關係是否前後矛盾  
    - 重要物件（武器、神器、機密檔案等）流向是否合理  
    - 世界觀規則（魔法、科技、政治制度等）是否自洽  
  - 團隊須在提案或審稿階段，輸出一份**可閱讀的衝突彙整報告**，輔助決策與溝通。

### 技術棧

- **框架**：Next.js（App Router）
- **樣式**：Tailwind CSS
- **UI 元件**：可搭配 shadcn/ui（依實作情況）
- **資料來源**：前端內建 mock data，無實際 LLM 推論流程

### 系統架構概念（Prototype 階段）

目前專案僅實作 **前端介面與資料展示層**，以利：

- 驗證使用流程與資訊層級設計
- 示範未來與後端 LLM / 邊緣節點串接時的**輸入 / 輸出介面契約**

在未來實作中，可於本專案基礎上：

- 將檔案上傳與版本資訊對接至實際後端服務
- 以 API 取得衝突偵測、角色一致性分析與知識圖譜資料
- 將本地／邊緣部署的 LLM 結果轉換為前端可視覺化呈現的結構化 JSON。

### 本機開發與執行方式

1. 安裝依賴套件：

```bash
npm install
```

2. 啟動開發伺服器：

```bash
npm run dev
```

3. 開啟瀏覽器並前往：

```text
http://localhost:3000
```

如有修改程式碼，頁面會自動熱重載。

以下僅列出與產品 Prototype 相關的主要目錄（實際結構可依最終程式為準）：

### 產品定位說明

- 這個專案主要用於 **展示敘事一致性檢核平台的操作介面與資訊架構**。
- 重點在於 UI 完整度、互動流程與資訊呈現，而非演算法或模型實作。
- 若要接上真實的 LLM 或邊緣部署環境，可在此 Prototype 基礎上替換 mock data 呼叫流程，並串接實際的後端 API。
